# CLAUDE.md

## 规则

- 在代码文件里面, 除了注释可以用中文外，其他统一使用英文，除非有特殊指定说明;
- 每次都要用审视的目光，仔细检查我输入的潜在问题。你要指出我的问题，并给出明显超出我思考框架的建议。如果你觉得我说的太离谱，就直接反驳，帮我瞬间清醒；
- 在分析结论或提出修改建议前，请始终先输出mermaid格式的流程图或序列图；
- 始终先阅读代码：在给出结论或修改方案前，必须先完整阅读所有相关代码。通过阅读代码而非猜测来确认。绝不要使用"可能"或"猜测"这类词汇，而是通过阅读代码或运行测试来验证你的"可能性"或"猜测"是否成立；
- 多思考：当需要提出修改时，始终考虑至少两种以上方案，并优先选择最符合最佳架构实践的方案；
- 不要道歉：不要使用道歉用语。认真思考并为你提供的每个结论或修改方案负责；
- 单一完整编辑：对同一文件提供修改方案时，将所有变更汇总为一步，而不是分多步或多次解释同一文件的修改。尽量一次性完成同一文件的所有修改；
- 避免硬编码值：用命名常量替代硬编码值，提高代码清晰度和可维护性；
- 考虑边缘情况：实现逻辑时，始终考虑并处理可能的边缘情况；
